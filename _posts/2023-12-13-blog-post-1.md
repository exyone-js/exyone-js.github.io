**linux.optools**

一套用于优化桌面和服务器 Linux 环境的 Shell 脚本集合。

**概述**

linux.optools 是一套全面的优化脚本集，旨在提升各种 Linux 发行版的性能、安全性和可用性。脚本按发行版和功能进行分类，便于您轻松找到并使用适合您系统的优化方案。

**功能****通用优化**

- **文件系统优化：**提升文件系统性能与可靠性
- **内存管理：**优化内存使用和交换行为
- **网络优化：**增强网络性能与安全性
- **进程优先级：**调整进程优先级以改善资源分配
- **安全强化：**采用最佳实践增强系统安全性
- **系统服务：**管理和优化系统服务

**发行版特定优化****Debian 系 (Debian, Ubuntu, Linux Mint)**

- 包管理优化
- 镜像源选择与更新
- Snap 包管理与优化

**Red Hat 系 (Fedora, CentOS, Rocky Linux)**

- DNF 包管理器优化
- SELinux 配置与安全强化
- Firewalld 配置
- 使用 tuned 配置文件优化服务

**Arch 系 (Arch Linux, Manjaro)**

- Pacman 包管理器优化
- AUR 支持与优化
- 使用 Earlyoom 改进 OOM 管理
- Btrfs 文件系统优化

**openSUSE (Leap, Tumbleweed)**

- Zypper 包管理器优化
- 使用 snapper 管理 Btrfs 快照
- 仓库管理 (Packman, 编解码器)
- 笔记本电脑电源管理

**开始使用****先决条件**

- 来自支持列表的 Linux 发行版
- Bash Shell (大多数 Linux 系统默认已安装)
- 运行大多数脚本需要 root 或 sudo 权限

## 前言

前段时间介绍各类邮箱时，我提到过**ProtonMail 隐私性好**——这似乎是很多人的共识。但最近我仔细体验了一番，发现它其实是个“雷”。

## 正文

### 隐私&开源？

首先，ProtonMail 确实是开源的，但只开源前端，**后端并不公开**。这意味着，我们根本不知道邮件在它的服务器上经历了什么处理。当然，一旦后端也开源，隐私性可能随着代码曝光而降低；可后端不开放，又无法保证服务器不会暗中上传数据。

这简直成了一个**循环论证**——所谓“完全隐私的邮箱”，根本就是个**伪命题**。

### 实力如何？

如果隐私性并不绝对，那我们再来看看它的其他功能：

免费版容量**只有 500MB**，这能装几个邮件？要是邮件带附件的话一百个件不到就用完了。免费用户只能注册一个地址，这点还算可以接受。付费后可以解锁*@pm.me这个“神级域名”。说实话，除非你特别想要pm.me这个域名，否则没必要升级会员。毕竟，这个域名确实简洁好看。

但问题是，ProtonMail 不支持国内常用支付方式。如果你费尽周折终于付了款，那为什么不直接自己去注册一个域名呢？有了域名，你就可以轻松设置域名邮箱，国内外很多邮局（比如QQ、Zoho）都支持。而 ProtonMail 的域名邮箱功能——同样需要付费升级。

对新手来说，我之前推荐过的Yandex也是不错的选择，它能自动帮你购买并配置.ru结尾的域名邮箱。

如果你拥有开放 25 端口且具备原生 IP 的云服务器，甚至可以自己搭建邮件系统（比如 Pmail）。这才是真正意义上的“完全隐私”——服务器和后端都在自己手里，不比那些空洞的宣传口号可靠多了？本站的合作伙伴酷鸭数据，联系客服或者提交工单即可申请开放25端口。

## 总结

ProtonMail的付费后体验相较于其它付费邮箱并不差，但是免费版体验极其糟糕。
